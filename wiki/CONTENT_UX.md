# [TASK: DesignSystem/Content UX | Standardize UI Copy And Labels | v0.1]

## Principles
- Clear over clever.
- Actionable verbs in buttons.
- Consistent terminology across apps.

## Labels
- Prefer concise noun labels for fields.
- Use helper text only when needed for decision clarity.

## Action Text
- Primary buttons: explicit verbs (`Save`, `Create`, `Confirm`).
- Secondary buttons: safe alternatives (`Cancel`, `Back`, `Close`).
- Destructive buttons: explicit consequences (`Delete`, `Remove`).

## Empty/Error States
- Explain what happened, why it matters, and next step.
- Avoid blame language.

## Confirmation UX
- High-risk actions need explicit confirmation language.
- Include object context in message where possible.
